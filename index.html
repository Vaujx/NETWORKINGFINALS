Acronyms in Networking

Understanding IP Addressing and Subnetting

Analogy: The Postal System

Think of IP addressing like the postal system:

  • The Network ID is like the city name
  • The Subnet is like a specific neighborhood within the city
  • The Host ID is like the specific house address within that neighborhood
  • The Subnet Mask is like the postal code system that helps determine which part of the address is the city/neighborhood and which part is the specific house

IP Address Classes

Class First Octet Range Default Subnet Mask Network Bits Host Bits
Class A 1-126 255.0.0.0 (/8) 8 24
Class B 128-191 255.255.0.0 (/16) 16 16
Class C 192-223 255.255.255.0 (/24) 24 8

Analogy: The Library System

IP classes are like a library's organization system:

  • Class A is like the main sections of the library (Fiction, Non-fiction, Reference) - few sections but each can hold many books
  • Class B is like the categories within each section (Mystery, Science Fiction, History) - more categories but each holds fewer books
  • Class C is like the specific shelves - many shelves but each can only hold a few books

Subnet Masks and Block Sizes

A subnet mask determines which portion of an IP address refers to the network and which portion refers to hosts.

Analogy: The Apartment Building

Think of a subnet mask like an apartment numbering system:

  • The 1's in the subnet mask are like the building and floor numbers (the network portion)
  • The 0's are like the individual apartment numbers on that floor (the host portion)
  • The more 1's you have in your subnet mask, the more buildings and floors you can have, but fewer apartments on each floor

Common Subnet Masks and Block Sizes

Subnet Mask CIDR Notation Block Size Binary (Last Octet)

Block Size Formula: Block Size = 256 - (Last octet of subnet mask)

Alternative Method: Count the number of host bits (0's) and calculate 2n

Subnetting Step-by-Step Process

Analogy: Dividing Land

Subnetting is like dividing a large piece of land into smaller plots:

  • The original network is like a large estate
  • Subnetting is like drawing boundaries to create neighborhoods
  • The subnet mask determines how many neighborhoods you can create and how many houses can fit in each neighborhood
  • The block size determines the "size" of each neighborhood

Step-by-Step Subnetting Process

Step 1: Determine the subnet mask and block size

  • Convert CIDR notation to decimal subnet mask
  • Calculate block size: 256 - (last octet of subnet mask)

Step 2: Find the network boundaries

  • Divide the IP space by the block size
  • Find which subnet range the IP belongs to

Step 3: Calculate key addresses

  • Network ID: First address in the subnet range
  • Broadcast address: Last address in the subnet range
  • First usable IP: Network ID + 1
  • Last usable IP: Broadcast address - 1

Step 4: Calculate subnets and hosts

  • Number of subnets = 2subnet bits
  • Number of hosts per subnet = 2host bits - 2

Subnetting Examples

Example 1: Finding Network Information for 192.160.1.10 /28

Step 1: Convert /28 to decimal subnet mask

/28 means 28 bits are used for the network portion

Binary: 11111111.11111111.11111111.11110000

Decimal: 255.255.255.240

Step 2: Calculate block size

Block size = 256 - 240 = 16

Or: Block size = 24 = 16 (4 host bits)

Step 3: Find network boundaries

Possible networks with block size 16:

  • 192.160.1.0
  • 192.160.1.16
  • 192.160.1.32
  • 192.160.1.48
  • ...

IP 192.160.1.10 falls between 0 and 16, so it belongs to 192.160.1.0 network

Step 4: Calculate key addresses

Network ID: 192.160.1.0

Broadcast Address: 192.160.1.15 (network ID + block size - 1)

First Usable IP: 192.160.1.1 (network ID + 1)

Last Usable IP: 192.160.1.14 (broadcast - 1)

Example 2: IP Address 220.100.1.9 with prefix /29

Step 1: Convert /29 to decimal subnet mask

/29 means 29 bits are used for the network portion

Binary: 11111111.11111111.11111111.11111000

Decimal: 255.255.255.248

Step 2: Calculate block size

Block size = 256 - 248 = 8

Or: Block size = 23 = 8 (3 host bits)

Step 3: Find network boundaries

Possible networks with block size 8:

  • 220.100.1.0
  • 220.100.1.8
  • 220.100.1.16
  • 220.100.1.24
  • ...

IP 220.100.1.9 falls between 8 and 16, so it belongs to 220.100.1.8 network

Step 4: Calculate hosts per subnet

Number of host bits = 32 - 29 = 3

Total hosts per subnet = 23 = 8

Usable hosts per subnet = 23 - 2 = 6

(We subtract 2 because we can't use the network ID and broadcast address as host addresses)

Step 5: Calculate total subnets

Class C base network has /24 prefix

Additional subnet bits = 29 - 24 = 5

Total subnets = 25 = 32

Example 3: Creating a subnet mask for 172.20.0.0 (900+ subnets with 50+ hosts each)

Step 1: Determine subnet bits needed

Need 900+ subnets → at least 10 subnet bits (210 = 1,024)

Class B starts with first 16 bits for network

First 16 + 10 additional subnet bits = /26

Step 2: Verify host requirements

Total bits = 32

Network bits = 26

Host bits = 32 - 26 = 6

Hosts per subnet = 26 - 2 = 62

62 > 50, so this meets our requirements

Step 3: Convert to subnet mask

Binary: 11111111.11111111.11111111.11000000

Decimal: 255.255.255.192

Packet Tracer Overview

Analogy: Network Simulation as a Virtual Lab

Packet Tracer is like having a complete networking laboratory on your computer:

  • It's like a "flight simulator" for network engineers
  • You can build and test networks without needing physical hardware
  • You can see how data moves through the network like watching a package delivery system

What is Packet Tracer?

  • A powerful network simulation program
  • Used to illustrate how computer networks work
  • Allows students to experiment with network behavior

Two Different Views

  1. Logical Workspace: Focus on logical connections and network design
  2. Physical Workspace: Focus on physical layout and equipment placement

Two Modes of Operation

  1. Real-time Mode: Network behaves as it would in real life
  2. Simulation Mode: Shows step-by-step movement of data packets through the network

Building Networks in Packet Tracer

Basic Steps to Create a Simple Network

  1. Add devices from the "End Devices" menu (like PCs)
  2. Configure device settings (name, IP address, etc.)
  3. Connect devices using the appropriate cables
    • Straight-through cables: Connect different types of devices (PC to switch)
    • Crossover cables: Connect similar devices (PC to PC)
  4. Test connectivity using Ping or PDU tools

Device Configuration Example

Configuring a PC

  1. Double-click the PC icon
  2. Change the device name (e.g., "IT 3A")
  3. Under Interface → FastEthernet, set IP address (e.g., 192.168.1.1)
  4. Set subnet mask (e.g., 255.255.255.0)

Testing Network Connectivity

Using the Simple PDU Tool

  1. Click on the "Add Simple PDU" tool
  2. Click on the source device
  3. Click on the destination device
  4. Check results in the bottom right corner

Simulation Mode Features

  • Capture/Forward Button: Steps through packet transmission
  • Reset Simulation: Restarts the simulation
  • Back Button: Rewinds the animation
  • Event List: Shows detailed packet information
  • PDU List Window: Displays information about all Protocol Data Units

Summary of Key Concepts

Subnetting Key Formulas

  • Block size = 256 - (last octet of subnet mask)
  • Number of subnets = 2subnet bits
  • Number of hosts per subnet = 2host bits - 2
  • Network ID = First address in block
  • Broadcast address = Last address in block (Network ID + block size - 1)
  • First usable IP = Network ID + 1
  • Last usable IP = Broadcast address - 1

Final Analogy: The Road System

The entire networking system is like a road system:

  • The Internet is like the highway system connecting cities
  • Networks are like cities
  • Subnets are like neighborhoods within cities
  • Host addresses are like individual house addresses
  • Subnet masks are like the rules that define city boundaries
  • Packet Tracer is like a city planning simulator where you can design and test your road system before building it
